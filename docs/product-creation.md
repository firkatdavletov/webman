# Добавление товаров

## Перед началом

Чтобы создать товар:

1. Откройте раздел `Каталог` → `Продукты`.
2. Нажмите `Добавить товар`.
3. Убедитесь, что в каталоге уже создана хотя бы одна категория.

Обязательные основные поля:

- `Название`.
- `Категория`.
- `Цена, руб.`: неотрицательное число.
- `Единица измерения`.
- `Шаг продажи`: положительное целое число.

Дополнительные поля:

- `Старая цена, руб.`: неотрицательное число или пустое поле.
- `Вес на витрине`.
- `SKU`.
- `Описание`.
- Группы модификаторов.

Флаг `Отображать на витрине` определяет, будет ли товар активен.

## Обычный товар без вариантов

Используйте обычный товар, если у него нет отдельных размеров, цветов или других исполнений с собственными SKU.

1. Заполните основные поля товара.
2. Оставьте флаг `Товар с вариантами` выключенным.
3. При необходимости добавьте группы модификаторов.
4. Заполните описание.
5. Нажмите `Создать товар`.

Пример:

| Поле | Значение |
| --- | --- |
| Название | Лимонад классический |
| Категория | Напитки |
| Цена | 250 |
| Единица измерения | шт |
| Шаг продажи | 1 |
| SKU | LEMONADE-CLASSIC |
| Товар с вариантами | Выключен |

После создания откроется страница товара. Фотографии самого товара загружаются на странице его деталей.

## Товар с вариантами

Используйте варианты, если исполнения товара отличаются по размеру, цвету или другой характеристике и должны иметь отдельные SKU.

### 1. Заполните основные поля

Заполните название, категорию, основную цену, единицу измерения и шаг продажи.

Основная цена обязательна даже для товара с вариантами. Цена конкретного варианта может быть указана отдельно или оставлена пустой.

SKU товара необязателен. Если он указан, он не должен совпадать с SKU любого варианта.

### 2. Включите режим вариантов

Включите флаг `Товар с вариантами`. Появятся разделы:

- `Опции товара`.
- `Варианты товара`.

### 3. Добавьте группы опций

Группа опций описывает характеристику, например цвет или размер.

1. Нажмите `Добавить опцию` или откройте автоматически созданную пустую группу.
2. Заполните:
   - `Code`: технический уникальный код группы, например `color` или `size`.
   - `Название`: отображаемое название, например `Цвет` или `Размер`.
   - `Sort order`: целое число для порядка отображения.
3. Добавьте хотя бы одно значение.
4. Для каждого значения заполните уникальный внутри группы `Code`, название и целочисленный `Sort order`.
5. Нажмите `Добавить опцию` или `Сохранить опцию`.

Пример групп:

| Группа | Code | Значения |
| --- | --- | --- |
| Цвет | color | `black`: Черный, `white`: Белый |
| Размер | size | `s`: S, `m`: M |

### 4. Добавьте варианты

1. Нажмите `Добавить вариант`.
2. Заполните обязательный `SKU`.
3. При необходимости заполните External ID, название, цену, старую цену, фотографии и `Sort order`.
4. Выберите ровно одно значение для каждой группы опций.
5. Нажмите `Добавить вариант`.
6. Повторите для остальных вариантов.

Каждый добавленный вариант должен выбрать значение из каждой группы. Добавлять все возможные комбинации необязательно.

Пример вариантов:

| SKU | Цвет | Размер | Цена |
| --- | --- | --- | --- |
| TSHIRT-BLACK-S | Черный | S | 1 500 |
| TSHIRT-BLACK-M | Черный | M | 1 500 |
| TSHIRT-WHITE-M | Белый | M | 1 600 |

SKU каждого варианта должен:

- быть заполнен;
- быть уникальным среди вариантов текущего товара;
- не совпадать с SKU самого товара;
- не использоваться другим товаром или вариантом в каталоге.

### 5. Сохраните товар

Нажмите `Создать товар`.

Система сначала сохраняет основные данные товара в выключенном состоянии, затем конфигурацию опций и вариантов. Если включен флаг `Отображать на витрине`, система включает товар только после успешного сохранения конфигурации вариантов. При успешном сохранении откроется страница деталей товара.

Если появилось сообщение, что основные данные сохранены, но создание товара не удалось завершить:

1. Не создавайте товар заново.
2. Исправьте ошибку в текущей форме.
3. Повторно нажмите `Создать товар`.

Форма повторно сохранит уже созданный товар и его конфигурацию вариантов. Заполненная форма и идентификатор уже сохраненного товара хранятся в черновике текущей вкладки; адрес страницы содержит идентификаторы только для проверки восстановления. Поэтому после обновления страницы повторное сохранение продолжит создание того же товара. Изменение идентификатора товара вручную в адресе не применяется и блокирует продолжение создания. Если браузер не смог сохранить последние изменения, на странице появится предупреждение и после обновления восстановится предыдущая сохраненная версия черновика. Если сам черновик недоступен, продолжение создания блокируется, чтобы не создать дубликат или не изменить другой товар. До успешного завершения товар остается выключенным и не отображается на витрине.

## Модификаторы

Модификаторы не являются вариантами.

- Вариант: отдельное исполнение товара с собственным SKU, например футболка размера M.
- Модификатор: дополнительный выбор покупателя, например соус или добавка.

Чтобы привязать модификатор:

1. Сначала создайте группу модификаторов в соответствующем разделе каталога.
2. В форме товара нажмите `Добавить модификатор`.
3. Выберите группу, задайте `Sort order` и состояние привязки.

Одну группу модификаторов нельзя привязать к товару несколько раз.

## Частые ошибки

### Нет доступных категорий

Сначала создайте хотя бы одну активную категорию.

### Не удается сохранить вариант

Проверьте:

- заполнен ли SKU варианта;
- уникален ли SKU;
- выбрано ли по одному значению каждой группы опций;
- существуют ли выбранные значения;
- являются ли цены неотрицательными числами;
- является ли `Sort order` целым числом.

### Не удается сохранить группу опций

Проверьте:

- заполнены ли `Code` и название;
- уникален ли `Code` группы;
- добавлено ли хотя бы одно значение;
- уникальны ли коды значений внутри группы;
- являются ли все `Sort order` целыми числами.

### Не удается сохранить фотографии варианта

Используйте изображения JPG, PNG или WEBP. Дождитесь окончания загрузки перед сохранением товара.
